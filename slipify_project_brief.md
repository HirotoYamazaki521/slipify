# slipify — プロジェクト概要

## プロジェクト概要

レシートや紙媒体をAIで読み取り、データとして蓄積・活用できるようにするアプリケーション。
まずはレシートのCSV出力から始め、段階的に対応範囲と出力形式を拡張していく。

---

## コアコンセプト

> 「撮って、貯めて、使う」
> 紙の情報をデジタル資産に変える

---

## 機能要件

### フェーズ1（MVP）
- レシート画像をアップロードする
- Claude APIで画像を解析し、構造化データ（店名・日付・品目・金額・合計など）を抽出する
- 抽出したデータをDBに保存する
- 保存済みデータをCSV形式でエクスポートできる
- CSVの出力フォーマット（カラム順・名称・区切り文字など）をカスタマイズできる

### フェーズ2（拡張）
- CSV以外の出力形式に対応する（Excel、JSON、PDF など）
- 読み取り対象を拡張する（レシート以外の紙媒体：請求書・名刺・契約書 など）
- 蓄積データの検索・フィルタリング・集計機能を追加する

---

## 技術要件

- **AI処理**: Claude API（画像読み取り・構造化データ抽出）
- **DB**: 未定（SQLite / PostgreSQL / Supabase などから選定）
- **プラットフォーム**: Webアプリ または ネイティブアプリ（未定）
- **CSVフォーマット**: ユーザーがカラム定義をカスタマイズ可能な設計にする

---

## 未決定事項（要検討）

| 項目 | 選択肢 | 備考 |
|------|--------|------|
| プラットフォーム | Webアプリ / モバイルアプリ | ユーザー層によって判断 |
| DBの種類 | SQLite / PostgreSQL / Supabase | スケール感次第 |
| 認証 | あり / なし | 複数ユーザー対応するか次第 |
| ホスティング | Vercel / Fly.io / その他 | Webアプリの場合 |

---

## 制約・方針

- Claude APIのキーはサーバー側で管理する（利用者はAPIキー不要）
- 将来的なサブスク課金モデルを想定した設計にする（使用量の記録を残す）
- 保守コストを低く保つシンプルな構成を優先する
- 仕様駆動開発（cc-sdd）で進める

---

## 将来的なビジョン

- レシートに限らず「あらゆる紙媒体をデータ化できるプラットフォーム」を目指す
- 対応書類の種類を増やしていく（請求書・領収書・名刺・契約書 etc）
- 出力フォーマットを増やしていく（CSV / Excel / JSON / PDF / 会計ソフト連携 etc）
